Communication

English will be the main language used for written communication and documentation
Communication is direct and open, it is important that everyone feels safe so we can avoid conflicts within the team
Technical arguments regarding code or art will be limited by setting up a pro/con list of each option which can be presented to the respective lead so they can make a final decision. This is to avoid long and undocumented discussions

Office Protocols
Working Hours
Outside of standard working hours, we use a clock-in/clock-out system as a substitute of stand ups/downs where you notify the team of the following:
When you start working + what you’re going to work on
When you stop working + what you’ve done
Example
Start workday: [Start Time]
[What you’re going to work on that day]
End workday: [End Time]
[What got done that day]
