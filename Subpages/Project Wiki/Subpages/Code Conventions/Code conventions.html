Code conventions

Line length

Code lines are not allowed to exceed 120 characters so that code will always be readable on most screen sizes.

File Names

Every file and folder name is written in UpperCamelCase.

Bracket Placement

Brackets are placed on a new line and not on the same line.
C#
Kopiëren
public class ExampleScript { }
​

Namespaces

The namespace name is written in UpperCamelCase and is based on the folder name the script is placed in. Every class needs to be inside of a namespace.
C#
Kopiëren
namespace ExampleNamespace { public class ExampleScript : MonoBehaviour { } }
​

Classes

The Class name is written in UpperCamelCase. If the function GetComponent is used to get a component from this GameObject you use RequireComponent above the class.
C#
Kopiëren
[RequireComponent(typeof(ExampleComponent))] public class ExampleScript : MonoBehaviour { private void Awake() { ExampleComponent exampleComponent = GetComponent<ExampleComponent>(); } }
​

Functions

The Function name is written in UpperCamelCase. Public functions need to have a summary.
C#
Kopiëren
private string ExampleFunction(string parameter) { string newString = $"test {paramter}"; return newString; } /// <summary> /// A summary explaining the function of the function. /// </summary> public void SecondExampleFunction { }
​
When there is only 1 line inside of an function you can use a lambda expression.
C#
Kopiëren
private void ExampleFunction() => SecondExampleFunction();
​

Variables

As a general rule of thumb almost all variables are private and when access is needed use a SerializeField or a Getter & Setter.
Private variable names always start with an '_' (Even when serialized) after which it is written in lowerCamelCase.
C#
Kopiëren
private Object _variableExample; [SerializeField] private Object _secondVariableExample;
​
Public variable names are written in lowerCamelCase.
C#
Kopiëren
public Object variableExample;
​
Readonly variable names are written the same as public variables so in lowerCamelCase.
C#
Kopiëren
public readonly Object variableExample;
​
Constant variable names are written in FULL_CAPITALS.
C#
Kopiëren
public const int EXAMPLE_CONSTANT_VALUE;
​
Internal variable names always start with 'i_' after which it is written in lowerCamelCase. And are never serializable in the inspector.
C#
Kopiëren
internal int i_variableExample;
​
Protected variable names always start with 'p_' after which it is written in lowerCamelCase. And are never serializable in the inspector.
C#
Kopiëren
protected int p_variableExample;
​
Temporary variables inside of an function always need to be written out and are written in lowerCamelCase.
C#
Kopiëren
private void ExampleFunction() { float temporaryFloat = 1f; int temporaryInt = 1; }
​
Temporary constants inside of an function always need to be written out and are written in FULL_CAPITALS.
C#
Kopiëren
private void ExampleFunction() { const float TEMPORARY_FLOAT = 1f; const int TEMPORARY_INT = 1; }
​
Lists are written the same as public or private variables and are always assigned in using 'new()'.
C#
Kopiëren
private List<GameObject> _exampleList = new();
​
Property names are written in UpperCamelCase.
C#
Kopiëren
public int ExampleInteger { get => _exampleInterger; set { if (value < 0) _exampleInterger = 0; } }
​

If statements

When there is only one line after an if statement it needs to be tabbed under the if statement and doesn't need brackets and same with the else.
C#
Kopiëren
if (_exampleBoolean) ExampleFunction(); else SeccondExampleFunction(); if (!_exampleBoolean) ThirdExampleFunction();
​
If either the if or the else in the statement contains multiple lines both the if and the else need brackets.
C++
Kopiëren
if (_exampleBoolean) { ExampleFunction(); } else { SeccondExampleFunction(); ThirdExampleFunction(); }
​

For loops

For loops are written using the same rules as if statements, When there is only 1 line inside of it it can be written without brackets.
C#
Kopiëren
for (int i = 0; i < _exampleList.Count; i++) ExampleFunction(_exampleList[i]); for (int i = 0; i < _exampleList.Count; i++) { ExampleFunction(_exampleList[i]); SecondExampleFunction(_exampleList[i]); }
​
For Each are written using the same rules as for loops.
C#
Kopiëren
foreach(GameObject gameObject in ExampleList) ExampleFunction(gameObject); foreach(GameObject gameObject in ExampleList) { ExampleFunction(gameObject); SecondExampleFunction(gameObject); }
​

Events

The Action name is written in UpperCamelCase and will almost always start with 'On'. Unity Events are written the same way as a Action.
C#
Kopiëren
public Action OnExampleEvent; public UnityEvent<int> OnSecondExampleEvent;
​

Structs

The struct name is written in UpperCamelCase and everything inside the struct follows the usual code conventions.
C#
Kopiëren
public struct ExampleStruct { public double x; public double y; }
​

Enums

The enum name is written in UpperCamelCase while the constants are in FULL_CAPITALS.
C#
Kopiëren
public enum ExampleEnum { FIRST_CONSTANT, SECOND_CONSTANT }
