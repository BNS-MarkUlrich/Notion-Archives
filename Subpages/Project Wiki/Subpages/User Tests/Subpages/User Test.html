Instructions

Record the User Test.
Gameplay recording
Face recording
Hand recording
Ask the player to talk as much as they can during their experience.
Decide and document when giving testers instructions. But by default give none.
Tip: When giving no instructions. Simply document observations, are they having trouble with a certain feature? Is something not clear? After a while and if the player does not start talking, ask questions about their current experience: "What do you think you need to do? What are you missing? What would you like to do in this situation? What would you change?" etc. Prepare these questions beforehand as much as possible.

Feedback

Note down feedback the player is giving during their experience. Additionally, Make sure to note down (and update) how many testers give this feedback.

Tip: Make sure to ask follow-up questions when the meaning of the player might be unclear. Testers tend to say things despite meaning or doing something else. Clearing this up is crucial!

Questions

Prepare questions before the user test curated to the goal of the user test. What are you testing and what is it you want to get out of the user test? Are you comparing to an older user test? AVOID yes/no questions whenever you can & try not to settle for yes/no answers when they are given. Ask follow-up questions whenever necessary!
Bad Example: "Did the bow feel nice?" Good Example: "How did you experience using the bow? What would you change?"

Conclusion

What is your plan of action based on the user test? What have you learned?
Based on feedback during Sprint 2 Review, the following conclusions are no longer entirely relevant. See Sprint 3 Planning and Sprint Board.
Add a mini-map for players to make it easier to navigate their surroundings and find where they are in the world
Add an outline to interactables when hovering over them to make it more clear that they can be interacted with
Shrink the grid to make sure players cannot access the outer border area of the map
Fix overlapping buildings and buildings on top of each other. Also make sure the construction preview cannot be interacted with by the player which can create floating buildings
Add more contrast in the green colours of each biome so they are more easily differentiated between
Give the wasteland biome a more unique colour, such as light/dark brown to create more unique colouring in the environment
Add UI and pictograms to more clearly indicate the usage of certain elements on the screen
Make sure the tower has a max range for detecting enemies so it does not fire outside of its max projectile range
Fix the grid updating in relation to placed buildings
Make sure interacting with objects is repeated automatically so you only have to click on an object once to start interacting with it
Add a threshold to the camera before it starts following the player to make it easier to navigate within smaller distances from the original location
Make sure the camera doesn’t trail or lean with the direction the tracked object is moving in
Make sure players are able to change their current zoom level within the game
Lastly, also keep in mind to note down positive feedback :)
