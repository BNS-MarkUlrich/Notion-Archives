Networking Workflow

How to Optimize a System for Multiplayer

NetworkBehaviour Inheritance: To equip a MonoBehaviour with network communication capabilities, make it inherit from the NetworkBehaviour class. This class provides the fundamental methods required for networking interactions.
Ownership Checks: Before allowing a user to manipulate a specific element or object, verify their ownership. Utilize the IsOwner boolean property to confirm whether a player has authority over a given asset.
Method Tagging: Structure your system efficiently using three primary method tags:
[ServerRpc]: Designate functions that transmit data to the server. For instance, this tag would be appropriate for relaying button presses for the server to process.
[Server]: Tag methods responsible for managing received data. Methods marked with [Server] execute exclusively on the server, ensuring that it maintains control.
[ObserversRpc]: This tag indicates methods used to broadcast updates from the server. When a change concerning a client occurs, this tag ensures all clients within the server are updated and visually synchronized.

public class PlayerMovement : NetworkBehaviour { public float speed = 5.0f; private void Update() { // Check if this client is the owner of this player so it can move it. if (!IsOwner) return; float horizontalInput = Input.GetAxis("Horizontal"); float verticalInput = Input.GetAxis("Vertical"); SendInputToServer(horizontalInput, verticalInput); } [ServerRpc] // ServerRpc sends data to the server private void SendInputToServer(float horizontalInput, float verticalInput) { // Send input to the server to set the position, this makes it so there is server authority which means players cannot set their own position. ProcessInput(horizontalInput, verticalInput); } [Server] // Server methods only get executed on server cleints. private void ProcessInput(float horizontalInput, float verticalInput) { // Server takes inputs and converts it into a position Vector3 movement = new(horizontalInput, 0, verticalInput); Vector3 scaledMovement = (float)TimeManager.TickDelta * speed * movement; transform.position += scaledMovement; BroadcastPosition(transform.position); } [ObserversRpc] // ObserversRpc wil send things to other clients in the server private void BroadcastPosition(Vector3 newPosition) { // new position gets broadcasted to all the other players in the server. transform.position = newPosition; } }

How to Test

Verify Network Prefab: Ensure the essential Network Manager prefab is integrated into the scene you intend to test.
Create a Build: Begin by generating a build of your project.
Run Multiple Instances: Launch the build at least three times:
Once to act as the dedicated server.
Twice to simulate two separate clients.
Comprehensive Testing: This configuration allows you to thoroughly test:
Data transmission from clients to the server.
Server processing and response.
Broadcasting of results to all connected clients.
