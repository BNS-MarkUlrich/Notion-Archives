Description

This is a document containing the API calls which can be made to the player data base to retrieve or post player data. We planned on using a cloud based storage service but in the end this solution didn't work. For more info go to "Player Data Storage."

Fetching Player Data

Purpose:
Retrieves a single player's data from the "LINX_PLAYER_DATA" database based on their ID.
Usage:
Method: GET
Base URL: an example URL:
http://localhost/get_player.php?id=123
Required Parameters:
id: The unique identifier of the player record to fetch.
Response:
Success:
HTTP Status: 200 OK
Content-Type: application/json
JSON object containing the following:
id
level
exp
buildings
Error:
JSON object with an error message:
"Player not found" (if no matching ID exists)
"Missing ID parameter" (if the 'id' parameter is not provided)

Updating Player Data

Purpose:
Modifies an existing player's data in the "LINX_PLAYER_DATA" database.
Handles updates to the 'level', 'exp', and 'buildings' fields of a player record.
Usage:
Method: GET
Base URL: an example URL:
http://localhost/update_player.php?id=123&level=5&exp=2000
Required Parameters:
id: The unique identifier of the player record to update.
Optional Parameters:
level: The new level value for the player (integer).
exp: The new experience value for the player (integer).
buildings: The updated buildings data for the player (likely a string representation of a data structure).
Response:
Success:
HTTP Status: 200 OK
Content-Type: application/json
JSON object containing the updated player data.
Error:
Text error message indicating the reason for failure.
Possible Security Improvements:
Authentication/Authorization: Implement a mechanism to restrict who can modify player data.
