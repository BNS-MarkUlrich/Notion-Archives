Description

This document details the functional and technical for the building system. A system that allows the player to place structures into the world to be gain an advantage in the game.

Functional Design

When the player starts the game they should immediately see the UI to use the building system. When opening the UI a few things should be shown to the player:

UI Clarity:

It should be clear to the player where the buttons are and what they do.
It should be clear that placing the structures requires resources gathered with the Resource Gathering .
To interact with the UI, the player should press the button for the structure they want to place and drag it into the physical world space. To be able to fully place an structure into the world a few things should be true:

Placable locations:

For a structure to be placed on a location the following thing need to be true:
There are no other structures in the way.
No resources should be in the way.
the slope should no be to steep.
Before a structure is fully placed, there should be a preview that first shows what you want to place. At this point the player should get 2 types of buttons:

Buttons:

A button to confirm the placement of the structure into the world.
A button to be able to rotate the structure around it's axis.
Deze afbeelding kon niet worden gevonden.

Technical Design

Pointer: To building system works together with the "Pointer", which is a system that sends a raycast from the camera and gets the info from that point in the world.

Setting position:

To get the position for the structure the Pointer is used to get the mouse position and if the layer of the ground allows the system to place a building there.
Check collision with the mouse position of the Pointer the function CheckPlaceable() is called. When this function is called it will loop through a list of already placed structures and compares the x and z bounds of both structures to see is these bounds overlap. If so than the bool _canPlace will be set to false.
Slope: If the layer is placable, the dot product will be calculated and used to check if the slope of the ground is not to steep to place anything, this value is give in the inspector.
If the slope is levelled, the collided ground is placable and the collider of the structure is not overlapping anything, then the preview let's the building be placed into the world.

Holding down mouse:

Through the Input Manager from Unity the actions of the mouse are send to the Building Manager and are bind to 2 action listeners, one for clicking the button down and one for releasing. Within these functions the bool _isHolding is set to detect when the mouse is held down.
The _isHolding bool is used inside the Update function to make sure that when the player holds the structure preview down it will move alongside with the mouse position on the screen.
