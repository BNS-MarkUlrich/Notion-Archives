Description

This document details the design choices for storing player data in our HTML5 game prototype. It outlines the need for persistent data storage to ensure progress is saved between sessions, fostering a more engaging player experience. The document explores various storage solutions, including external servers, cloud platforms (AWS), and local hosting.

Functional Design

To accommodate the short session nature of HTML5 games and player preferences, we wanted to add a mechanism to save player data, including level progress, across sessions. This ensures that when a player quits the game and returns later, their progress is retained, creating a more engaging and seamless gameplay experience.
